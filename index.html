What are regular expressions?

A regular expression, regex or regexp is a sequence of characters that define a search pattern. Usually such patterns are used by string searching algorithms for "find" or "find and replace" operations on strings.

Many programming languages provide regex capabilities either built-in or via libraries.

In JavaScript, regular expressions are also objects. You construct a regular expression in one of two ways:

  1. Using a regular expression literal, which consists of a pattern enclosed between slashes, as follows: 
    var re = /ab+c/;
  2. Or calling the constructor function of the RegExp object, as follows: 
    var re = new RegExp('ab+c');

Basic concepts and syntax

RegExp is a pattern of special characters used to match strings in a search, typically made up from special characters called metacharacters
Metacharacter Mathes
. Any one character, except new line
[] Any one of the enclosed characters (e.g. a-z)
* Zero or more of preceding character
? or \? Zero or one of the preceding characters
+ or \+ One or more of the preceding characters
Metacharacter Matches
| Either the expression before or the expression after the operator.
^ Beginning of line
$ End of line
\char Escape the meaning of char following it
[^] One character not in the set
{m,n} Preceding element at least m and not more than n times.

Character Classes

Character Class Matches
\d Digits
\D Non-digits
\s Space symbols, tabs, newlines
\S All but \s
\w English letters, digits, underscore '_'
\W All but \w

Flags

Flag Matches
i Case-insensitive
g All matches
m Multiline mode
s Allows . to match newlines
u Enables full unicode support
y Sticky mode

Methods of RegExp and string

The global RegExp object has no methods of its own; however, it does inherit some methods through the prototype chain.

str.search(reg)

Returns the position of the first match or -1 if none found:


let str = "A drop of ink may make a million think";
alert( str.search( /a/i ) ); // 0 (first match at zero position)

The important limitation: search only finds the first match.

str.match(reg), no “g” flag

Looks for the first match only. The result is an array with that match and additional properties:

  • index – the position of the match inside the string,
  • input – the subject string.

let str = "Fame is the thirst of youth";

let result = str.match( /fame/i );

alert( result[0] );    // Fame (the match)
alert( result.index ); // 0 (at the zero position)
alert( result.input ); // "Fame is the thirst of youth" (the string)

str.match(reg) with “g” flag

Returns an array of all matches


let str = "HO-Ho-ho!";

let result = str.match( /ho/ig );

alert( result ); // HO, Ho, ho (array of 3 matches, case-insensitive)

If there are no matches, the result is not an empty array, but null

str.split(regexp|substr, limit)

Splits the string using the regexp (or a substring) as a delimiter.

limit - Integer specifying a limit on the number of splits to be found. Optional.


alert('12-34-56'.split(/-/)) // array of [12, 34, 56]

str.replace(str|reg, str|func)

When the first argument of replace is a string, it only looks for the first match, but a regexp with an obligatory gflag looks for all matches 


alert( '12-34-56'.replace( /-/g, ":" ) )  // 12:34:56

regexp.exec(str)

If there’s no g, then regexp.exec(str) returns the first match, exactly as str.match(reg). But if there’s g, then:

  • regexp.exec(str) returns the first match and remembers the position after it in regexp.lastIndex property.
  • The next call starts to search from regexp.lastIndex and returns the next match.
  • If there are no more matches then regexp.exec returns null and regexp.lastIndex is set to 0.

MatchAll does the same, at least for modern browsers. But what matchAll can’t do – is to search from a given position. Let’s search from position 13. What we need is to assign regexp.lastIndex=13 and call regexp.exec: 


let str = "A lot about JavaScript at https://javascript.info";

let regexp = /javascript/ig;
regexp.lastIndex = 13;

let result;

while (result = regexp.exec(str)) {
	alert( `Found ${result[0]} at ${result.index}` );
	// shows: Found javascript at 34
}

regexp.test(str)

The method regexp.test(str) looks for a match and returns true/false whether it finds it. 


let str = "I love JavaScript";
alert( /random/i.test(str) ); // false
alert( /love/i.test(str) ); // true

If the regexp has 'g' flag, then regexp.test advances regexp.lastIndex property, just like regexp.exec. 


let regexp = /love/gi;

let str = "I love JavaScript";

// start the search from position 10:
regexp.lastIndex = 10
alert( regexp.test(str) ); // false (no match)
